# Doc Image Maintainer

## 目标

将文档图片资源整理为可维护结构，并确保文档引用不失效：

1. 将图片迁移到“相对当前 md 文件目录”的 `./imgs`。
2. 将无语义命名（如 `image.png`、`image-1.png`）重命名为可追踪命名。
3. 删除未被任何文档引用的图片文件。
4. 完成后做引用完整性校验。

## 工作流

1. 盘点图片与引用

- 枚举目标目录下图片文件（常见扩展名：png/jpg/jpeg/gif/svg/webp）。
- 在 `docs/src` 全量扫描引用（至少覆盖 Markdown 图片 `![...](...)` 与 HTML `<img src="...">`）。

2. 迁移到相对 `./imgs`

- 对每个文档单独处理：迁移目标固定为“该 md 同级目录下的 `imgs` 子目录”。
- 示例：`docs/src/qiankun/asset-path.md` 的图片目标目录是 `docs/src/qiankun/imgs`。

3. 统一重命名

- 对无语义旧名（如 `image*.png`）重命名为：`<md-basename>-figure-XX.<ext>`。
- 示例：`asset-path.md` -> `asset-path-figure-01.png`。
- 若目录已有同名文件，递增编号避让，不覆盖已有文件。

4. 改写文档引用

- 只改写真实图片引用，不改正文说明文本、代码块、示例代码字符串。
- 引用路径统一使用相对写法（如 `./imgs/asset-path-figure-01.png`）。

5. 清理未引用图片

- 迁移与改写后，再次全量扫描 `docs/src`。
- 删除“未被任何文档引用”的图片文件。
- 删除前先确认不是代码示例文本误判（例如文中的 `url(./image.png)` 说明文本不算图片引用）。

6. 最终校验

- 确认不存在悬空引用（文档引用的文件不存在）。
- 确认目标 md 所在目录不再残留应迁移的根目录图片。
- 输出变更摘要：迁移、重命名、删除、引用更新条目。

## 执行约束

- 优先使用 `rg` 做搜索与校验。
- 仅在用户指定范围内改动，避免跨目录误操作。
- 删除操作前先完成引用改写与二次扫描，避免误删。
- 如有命名冲突、同名跨目录引用或来源不明确，先保守处理并明确说明假设。
